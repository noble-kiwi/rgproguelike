# RPG Roguelike

Курсовой проект — двухмерная action-RPG в жанре roguelike на C++17 с использованием
библиотеки SFML 2.6.

## Возможности

- Игровой цикл с delta time, окно 1280×720, стабильные 60 FPS
- Игрок: движение в 8 направлениях (WASD/стрелки), стрельба в курсор (ЛКМ),
  рывок/dash (Space/Shift) с коротким иммунитетом и откатом ~1 сек
- Характеристики игрока: HP, урон, скорость, скорострельность — улучшаются
  подобранными предметами
- 4 типа противников:
  - **Слизень** — движется к игроку, урон при касании
  - **Стрелок** — держит дистанцию, периодически стреляет снарядами
  - **Берсерк** — быстро бросается к игроку, высокий урон в рывке
  - **Босс** — несколько фаз атаки (меняются по HP), в финальной комнате
- Процедурная генерация этажа: 6–10 случайно соединённых комнат
  (север/юг/запад/восток) с типами: стартовая, обычная (с врагами),
  с предметом, магазин, финальная (босс)
- Дверь в следующую комнату открывается только после гибели всех врагов
- 2 этажа с нарастающей сложностью
- Предметы: аптечка, апгрейд урона, апгрейд скорости, щит, монеты
- Магазин: покупка предметов за монеты (выбиваются из врагов)
- HUD: полоска здоровья с числовым значением, счёт, рекорд, этаж,
  количество монет, иконки активных предметов, миникарта этажа
- Экраны: главное меню (с отображением рекорда), пауза (Esc),
  Game Over, экран Победы
- Сохранение лучшего счёта в файл `highscore.txt`

## Управление

| Клавиша | Действие |
|---|---|
| `WASD` / стрелки | движение в 8 направлениях |
| `ЛКМ` | атака в направлении курсора |
| `Space` / `Shift` | рывок (dash) |
| `E` | взаимодействие (покупка в магазине) |
| `Esc` | пауза / выход в меню |
| `Enter` / `Space` | подтверждение в меню |

## Архитектура

Классы (`src/`):

| Класс | Назначение |
|---|---|
| `AssetManager` | Singleton — загрузка и хранение шрифтов |
| `Entity` | Абстрактный базовый класс для сущностей (HP, позиция, скорость) |
| `Player` | Игрок: ввод, атака, dash, инвентарь (наследует `Entity`) |
| `Enemy` | Противники всех типов с собственным ИИ (наследует `Entity`) |
| `Projectile` | Снаряды игрока и врагов |
| `Item` | Подбираемые предметы и товары магазина |
| `Room` | Комната: тайловая сетка, двери, коллизии |
| `Level` | Этаж: граф комнат, процедурная генерация, переходы |
| `HUD` | Heads-Up Display: HP bar, счёт, миникарта, иконки |
| `Game` | Главный контроллер: состояния (меню/игра/пауза/Game Over/Победа), игровой цикл |

Управление памятью: RAII + `std::unique_ptr` для полиморфных врагов,
`std::vector` для коллекций.

## Сборка (Windows)

Требования:
- Windows 10 / 11
- CMake 3.16+
- Visual Studio 2019/2022 с компонентом «Desktop development with C++» (MSVC)

SFML 2.6.1 подтягивается автоматически через CMake FetchContent и линкуется
статически — никаких отдельных `.dll` рядом с `.exe` класть не нужно.

### Сборка из командной строки (Developer Command Prompt for VS)

```bat
cmake -S . -B build
cmake --build build --config Release
build\Release\RPG_Roguelike.exe
```

### Сборка в Visual Studio

1. Открыть папку проекта через «Open a local folder».
2. VS сам обнаружит `CMakeLists.txt` и сгенерирует конфигурацию.
3. Выбрать конфигурацию `x64-Release` и запустить цель `RPG_Roguelike`.

### Сборка в CLion

1. Открыть папку как CMake-проект.
2. Toolchain — «Visual Studio» (amd64).
3. Профиль — `Release`, запустить `RPG_Roguelike`.

│   ├── Item.{h,cpp}         # предметы
│   ├── Room.{h,cpp}         # комната (тайловая сетка)

## Ресурсы (шрифты)

Положите файл `font.ttf` в папку `assets/`. Рекомендуется шрифт с поддержкой
кириллицы — например, [DejaVu Sans](https://dejavu-fonts.github.io/) или
[Noto Sans](https://fonts.google.com/noto/specimen/Noto+Sans) (переименовать в
`font.ttf`).

`AssetManager` ищет шрифт по путям:

1. `assets/font.ttf` (рядом с `.exe` — CMake сам копирует папку после сборки)
2. `../assets/font.ttf` (если запуск идёт из подпапки `build/Release/`)

Если шрифт не найден — игра запустится, но текст HUD и меню не будет
отображаться.
